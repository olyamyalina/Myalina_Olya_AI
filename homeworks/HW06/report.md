## 1. Dataset

- Выбран датасет: `S06-hw-dataset-02.csv`
- Размер: 18 000 строк, 39 столбцов (включая `id` и `target`)
- Целевая переменная: `target`
  - класс 0: ~73.7%
  - класс 1: ~26.3%
- Признаки:
  - все признаки числовые;
  - присутствуют дополнительные признаки с нелинейными взаимодействиями (`x_int_1`, `x_int_2`);
  - столбец `id` исключён из признаков и не использовался при обучении моделей.

Датасет представляет собой задачу бинарной классификации с умеренным дисбалансом классов.

---

## 2. Protocol

- Разбиение данных:
  - train / test = 80% / 20%;
  - `random_state = 42`;
  - использовалась стратификация по таргету (`stratify=y`), чтобы сохранить распределение классов.
- Подбор гиперпараметров:
  - выполнялся **только на train** с помощью кросс-валидации;
  - использовалась `GridSearchCV` с 5 фолдами;
  - оптимизировалась метрика ROC-AUC (задача бинарная).
- Метрики качества:
  - `accuracy` — базовая метрика точности;
  - `f1` — учитывает дисбаланс классов;
  - `ROC-AUC` — основная метрика, отражает способность модели различать классы независимо от порога.

---

## 3. Models

В ходе работы были обучены и сравнены следующие модели:

- **DummyClassifier**  
  Стратегия `most_frequent`, использовался как baseline.

- **LogisticRegression**  
  Пайплайн с `StandardScaler`, линейная baseline-модель.

- **DecisionTreeClassifier**  
  Контроль сложности через подбор:
  - `max_depth`
  - `min_samples_leaf`

- **RandomForestClassifier**  
  Ансамбль деревьев (bagging), гиперпараметры:
  - `n_estimators`
  - `max_depth`
  - `min_samples_leaf`
  - `max_features`

- **GradientBoostingClassifier**  
  Последовательный ансамбль деревьев (boosting), гиперпараметры:
  - `n_estimators`
  - `learning_rate`
  - `max_depth`

StackingClassifier не использовался.

---

## 4. Results

Финальные метрики качества на тестовой выборке:

| Model              | Accuracy | F1     | ROC-AUC |
|--------------------|----------|--------|---------|
| DummyClassifier    | 0.7375   | 0.0000 | 0.5000  |
| LogisticRegression | 0.8119   | 0.5607 | 0.7977  |
| DecisionTree       | 0.8383   | 0.6576 | 0.8371  |
| RandomForest       | 0.8872   | 0.7466 | 0.9258  |
| GradientBoosting   | 0.9006   | 0.7947 | **0.9271** |

Лучшая модель по ROC-AUC — **GradientBoostingClassifier**.

---

## 5. Analysis

- **Устойчивость**:  
  GradientBoosting показывает высокое качество и стабильность за счет последовательного уменьшения ошибок предыдущих деревьев.

- **Ошибки**:  
  Confusion matrix для GradientBoosting показывает хорошее разделение классов и существенно меньше ошибок по сравнению с baseline.

- **Интерпретация**:  
  Рассчитана permutation importance для GradientBoosting.  
  Наиболее важные признаки вносят существенный вклад, подтверждая способность ансамбля эффективно использовать нелинейные взаимодействия.

---

## 6. Conclusion

- Одиночные деревья решений склонны к переобучению и требуют контроля сложности.
- Ансамбли (RandomForest, GradientBoosting) значительно превосходят одиночные модели по качеству.
- Bagging снижает variance и повышает стабильность.
- Boosting показывает сильные результаты, но чувствителен к подбору гиперпараметров.
- ROC-AUC — оправданная основная метрика для бинарной классификации с дисбалансом классов.
- Честный ML-протокол (фиксированный train/test, CV только на train) обеспечивает корректное сравнение моделей.
