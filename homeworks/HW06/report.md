## 1. Dataset

- Выбран датасет: `S06-hw-dataset-02.csv`
- Размер: 18 000 строк, 39 столбцов (включая `id` и `target`)
- Целевая переменная: `target`
  - класс 0: ~73.7%
  - класс 1: ~26.3%
- Признаки:
  - все признаки числовые;
  - присутствуют дополнительные признаки с нелинейными взаимодействиями (`x_int_1`, `x_int_2`);
  - столбец `id` исключён из признаков и не использовался при обучении моделей.

Датасет представляет собой задачу бинарной классификации с умеренным дисбалансом классов.

---

## 2. Protocol

- Разбиение данных:
  - train / test = 80% / 20%;
  - `random_state = 42`;
  - использовалась стратификация по таргету (`stratify=y`), чтобы сохранить распределение классов.
- Подбор гиперпараметров:
  - выполнялся **только на train** с помощью кросс-валидации;
  - использовалась `GridSearchCV` с 5 фолдами;
  - оптимизировалась метрика ROC-AUC (так как задача бинарная).
- Метрики качества:
  - `accuracy` — базовая метрика точности;
  - `f1` — важна при наличии дисбаланса классов;
  - `ROC-AUC` — основная метрика для бинарной классификации, отражающая способность модели различать классы независимо от порога.

---

## 3. Models

В ходе работы были обучены и сравнены следующие модели:

- **DummyClassifier**  
  Использовался в качестве baseline (стратегия `most_frequent`).

- **LogisticRegression**  
  Реализована в пайплайне с `StandardScaler`. Использовалась как сильный линейный baseline.

- **DecisionTreeClassifier**  
  Проводился контроль сложности модели. Подбирались гиперпараметры:
  - `max_depth`
  - `min_samples_leaf`

- **RandomForestClassifier**  
  Ансамбль деревьев решений (bagging). Подбирались:
  - `n_estimators`
  - `max_depth`
  - `min_samples_leaf`
  - `max_features`

- **GradientBoostingClassifier**  
  Последовательный ансамбль деревьев (boosting). Подбирались:
  - `n_estimators`
  - `learning_rate`
  - `max_depth`

StackingClassifier не использовался.

---

## 4. Results

Финальные метрики качества на тестовой выборке:

| Model              | Accuracy | F1     | ROC-AUC |
|--------------------|----------|--------|---------|
| DummyClassifier    | 0.7375   | 0.0000 | 0.5000  |
| LogisticRegression | 0.8119   | 0.5607 | 0.7977  |
| DecisionTree       | 0.8383   | 0.6576 | 0.8371  |
| RandomForest       | 0.8919   | 0.7600 | **0.9287** |
| GradientBoosting   | 0.9006   | 0.7947 | 0.9271  |

Победителем была выбрана модель **RandomForestClassifier**, так как она показала наилучшее значение ROC-AUC — основной метрики для данной задачи.

---

## 5. Analysis

- **Устойчивость**:  
  Модель RandomForest демонстрирует стабильное качество за счёт усреднения большого числа деревьев, что снижает variance по сравнению с одиночным деревом решений.

- **Ошибки**:  
  Confusion matrix для лучшей модели показывает хорошее разделение классов и существенно меньшее количество ошибок по сравнению с baseline-моделями.

- **Интерпретация**:  
  Для лучшей модели был рассчитан permutation importance.  
  Наиболее важные признаки вносят существенный вклад в качество предсказаний, что подтверждает способность ансамбля эффективно использовать нелинейные взаимодействия между признаками.

---

## 6. Conclusion

- Одиночные деревья решений склонны к переобучению и требуют явного контроля сложности.
- Ансамбли (Random Forest, Gradient Boosting) значительно превосходят одиночные модели по качеству.
- Bagging позволяет эффективно снижать variance и повышать стабильность модели.
- Boosting даёт сильные результаты, но может быть чувствителен к настройке гиперпараметров.
- Использование ROC-AUC как основной метрики для бинарной классификации с дисбалансом является оправданным.
- Честный ML-протокол (фиксированный train/test и CV только на train) позволяет корректно сравнивать модели.
